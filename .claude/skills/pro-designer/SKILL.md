# Pro Designer

## Visual Hierarchy

1. **Size**: Larger elements draw attention first
2. **Color**: High contrast and brand colors for emphasis
3. **Spacing**: Whitespace groups related elements
4. **Typography**: Weight and size establish hierarchy

## Accessibility (WCAG)

- **Color contrast**: Minimum 4.5:1 for normal text, 3:1 for large text
- **Touch targets**: Minimum 44x44px for interactive elements
- **Focus states**: Visible focus indicators for keyboard navigation
- **Alt text**: Descriptive alternatives for images
- **Semantic HTML**: Use proper heading hierarchy (h1 > h2 > h3)

## Layout Principles

- **F-pattern**: Users scan left-to-right, top-to-bottom
- **Progressive disclosure**: Show essential info first, details on demand
- **Consistent spacing**: Use 4px or 8px base unit
- **Breathing room**: Don't crowd elements; whitespace improves readability

## Typography

- **Line length**: 50-75 characters for body text
- **Line height**: 1.5-1.75 for body text, 1.2-1.3 for headings
- **Font sizes**: Establish clear hierarchy (e.g., 14px, 16px, 20px, 24px, 32px)

## Color Usage

- **Primary**: Main actions, brand identity
- **Secondary**: Supporting elements
- **Neutral**: Text, backgrounds, borders
- **Semantic**: Success (green), warning (amber), error (red), info (blue)

## Interaction Design

- **Feedback**: Every action should have visible feedback
- **Loading states**: Show progress for async operations
- **Error states**: Clear, actionable error messages
- **Empty states**: Guide users when no content exists

## Best Practices

- Design mobile-first, enhance for larger screens
- Maintain consistent component styling throughout
- Test with real content, not lorem ipsum
- Consider edge cases: long text, missing images, empty states
